# Release Plan

## Versioning
- Semantic versioning: MAJOR.MINOR.PATCH.

## Release Criteria
- Builds run crash-free; startup stable across OS matrix.
- Feature acceptance checklists met; docs updated.
- Tests pass; performance thresholds met.

## Checklists
- QA: install, launch, basic workflows.
- Docs: roadmap, plan, features status updated.
